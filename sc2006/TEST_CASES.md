# Test Cases

This document lists black-box and white-box test cases for the main app flows. It is meant to be a starting point for Jest tests and manual QA.

## Notes

- Black-box tests validate user-visible behavior.
- White-box tests validate internal branches, edge cases, and database-driven logic.
- The white-box cases above can be converted into unit tests or API route tests with Jest and mocks for Prisma, cookies, and fetch.
- Actual Result values are populated from executed Jest checks.
